# KiKi Fragancia — Design Improvement Plan

Generated by /plan-design-review · 2026-05-17 · Branch: main · Review score: 5/10 → 8/10

## Overview

Site is a Venezuelan luxury perfume e-commerce. Static, no backend. WhatsApp-driven purchasing.
Stack: React 19 + Vite 8 + Tailwind v4 + React Router v7.

**Classifier:** MARKETING/LANDING PAGE (root `/`) + APP UI (Tienda) + Hybrid (ProductDetail)

## Decisions Made (12 total)

### D2 — BrandStory: Ocultar imagen placeholder hasta tener foto real
**Decision:** Ocultar el div `brand-story-img-wrap` completamente.
**Why:** El placeholder muestra texto de desarrollador "brand-story.jpg lifestyle photo perfume editorial" visible a todos los usuarios. Rompe la credibilidad justo en la sección de confianza.
**File:** `src/components/BrandStory.jsx`
**Action:** Eliminar el bloque `{/* Right: image */}` completo (div `brand-story-img-wrap`). La sección queda 1-columna. Cuando haya foto real, restaurar con `<img>` y `alt` descriptivo.

### D3 — Inconsistencia de seguidores: Unificar en 110k+
**Decision:** Cambiar InstagramFeed badge de "33k+" a "110k+" (el número real de la cuenta).
**Why:** BrandStory muestra "110k seguidores" (animado), InstagramFeed muestra "33k+". Inconsistencia que rompe confianza.
**Files:** `src/components/InstagramFeed.jsx:111`
**Action:** Cambiar `33k+` a `110k+` en el badge de InstagramFeed.

### D4 — Botón carrito: Verde #25D366 en estado "added"
**Decision:** Cuando `added === true`, el botón de "Agregar al carrito" cambia a fondo `#25D366` (verde WA).
**Why:** Actualmente usa `var(--gold)` en ambos estados — el usuario no tiene feedback visual de que la acción ocurrió.
**File:** `src/pages/ProductDetail.jsx:256-260`
**Action:**
```jsx
background: added ? '#25D366' : 'var(--gold)',
color: added ? '#FFF' : '#0A0A0A',
```
El color vuelve a gold después de 1.8s cuando `added` vuelve a false.

### D5 — Tienda empty state: Mejorar con icono, texto cálido y 2 CTAs
**Decision:** Cuando `filtered.length === 0`, mostrar un estado con más calidez.
**Why:** "Sin resultados para esos filtros" + "Limpiar filtros →" es funcional pero frío. En e-commerce el empty state es una oportunidad de retener al usuario.
**File:** `src/pages/Tienda.jsx:423-441`
**Action:** Reemplazar el empty state con:
```jsx
<div style={{ textAlign: 'center', padding: '80px 20px' }}>
  {/* SVG de perfume/frasco pequeño, ~40px, en gold tenue */}
  <p style={{ fontFamily: "'Cormorant Garamond', serif", fontSize: '26px',
    color: 'rgba(250,250,248,0.45)', fontStyle: 'italic', marginBottom: 8 }}>
    No encontramos esa fragancia
  </p>
  <p style={{ fontFamily: "'DM Sans', sans-serif", fontSize: '13px',
    color: 'rgba(250,250,248,0.25)', marginBottom: 28 }}>
    228 fragancias esperan — intenta con otros filtros
  </p>
  <div style={{ display: 'flex', gap: 12, justifyContent: 'center', flexWrap: 'wrap' }}>
    <button onClick={clearFilters} style={{ /* gold outlined */ }}>Ver todas</button>
    <a href="https://wa.me/584120221983" style={{ /* gold filled */ }}>Consultar por WhatsApp</a>
  </div>
</div>
```

### D6 — CartDrawer: Cerrar + toast verde al abrir WhatsApp
**Decision:** Al hacer click en "Consultar por WhatsApp" en CartDrawer, cerrar el drawer y mostrar un toast verde durante 3 segundos.
**Why:** El drawer permanece abierto después del click — el usuario no sabe si la acción fue completada.
**File:** `src/components/CartDrawer.jsx`
**Action:**
1. En el handler del botón WA: llamar `onClose()` después de `window.open(waUrl)`
2. Agregar un toast global (puede ser un `<div>` fixed en `App.jsx` o dentro del CartDrawer)
3. Toast: fondo `#25D366`, texto "¡Listo! Te responderemos pronto", 3 segundos, fade out

### D7 — Pilares Guarantee: Mantener estilo minimal (sin iconos)
**Decision:** No agregar iconos a los 3 pilares. El estilo minimal gold-line es intencional.
**Rationale:** Consistente con el dark luxury del sitio. Iconos añadirían ruido visual.

### D8 — Crear DESIGN.md
**Decision:** Crear `DESIGN.md` en la raíz del proyecto.
**Why:** Las decisiones de diseño viven en código y CONTEXT.md (que está en .gitignore). Sin DESIGN.md, se repiten errores como revertir el Guarantee a ivory.
**Content mínimo:** paleta de color con hex exactos, tipografía, reglas de tema (dark vs ivory), componentes clave, reglas críticas (gold correcto, no framer-motion en landing, Guarantee dark, excepción CSS de InstagramFeed).

### D9 — InstagramFeed Tailwind: Documentar como excepción conocida
**Decision:** No migrar InstagramFeed a kiki CSS classes ahora. Documentar en DESIGN.md como excepción conocida.
**Why:** El componente funciona. Riesgo de migración > ganancia. Excepción clara.

### D10 — aria-hidden en elementos decorativos
**Decision:** Agregar `aria-hidden="true"` al `<video>` en Hero, al `<canvas>` en VaporCanvas, y al `<canvas>` en CursorTrail.
**Why:** Elementos 100% decorativos que lectores de pantalla intentan anunciar.
**Files:**
- `src/components/Hero.jsx:34` — `<video ... aria-hidden="true">`
- `src/components/VaporCanvas.jsx` — `<canvas ... aria-hidden="true">`
- `src/components/CursorTrail.jsx` — `<canvas ... aria-hidden="true">`

### D11 — Fotos de productos: No es un issue
**Finding:** 228/228 productos tienen foto. El placeholder de texto nunca aparece en producción.

### D12 — Instagram API: Diferir para el futuro
**Decision:** Mantener el feed de Instagram hardcodeado. Documentar como deuda futura en TODOS.md.
**Why:** Integrar Instagram Basic Display API requiere backend (Vercel function) + token que expira cada 60 días. Demasiado esfuerzo para la fase actual.

## TODOS.md (deuda futura)

**Instagram API Real:**
- Qué: Integrar Instagram Basic Display API para mostrar posts reales
- Por qué: El feed actual es hardcodeado con fotos del catálogo. Los posts reales de Instagram tienen más prueba social.
- Pros: Feed siempre actualizado sin tocar código
- Contras: Requiere Vercel function, token OAuth que expira cada 60 días, renovación automática. Complejidad alta.
- Depende de: Tener un backend (Vercel function) funcional
- Estimado: ~1 día human, ~45min CC

**Foto real para BrandStory:**
- Qué: Agregar una foto editorial lifestyle de perfumes para la sección "Nuestra historia"
- Por qué: La imagen está oculta ahora (D2), pero la sección podría tener más impacto visual con una foto real
- Pros: Sección más impactante, más personal, humaniza la marca
- Contras: Requiere sesión fotográfica o comprar foto de stock premium
- Estimado: Fotografía (variable), implementación ~10min CC
